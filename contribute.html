How to Contribute

Help grow the 4n6 forensics cheat sheet by adding new tools, techniques, and references.

1 Fork & Clone

Fork the repository on GitHub and clone it locally:

git clone https://github.com/thureinoo/4n6.git
cd 4n6

2 Add Your Entry

Open js/data.js and add a new entry object to the ENTRIES array. Follow this template:

{
  title: "Tool Name - What It Does",
  description: "Brief description of the tool or technique",
  category: ["Memory"],
  phase: ["Analysis"],
  os: ["Linux", "Windows"],
  tools: ["ToolName"],
  commands: [
    { label: "Basic usage", cmd: "tool -f {DUMP_FILE} --option" },
    { label: "Advanced", cmd: "tool -f {FILE} --verbose --output result.txt" }
  ],
  references: ["https://example.com/docs"]
}

3 Use Valid Values

Category:

Crypto/Encoding Disk File Analysis Logs Malware Memory Metadata Network Registry Stego References

Phase:

Acquisition Analysis Carving Recovery Timeline Triage

OS:

Linux Windows macOS Android iOS

4 Use Placeholders

Use these placeholders in commands so they auto-replace with user input:

  • {DUMP_FILE} — Memory dump file path
  • {IMAGE} — Disk image file path
  • {PCAP} — Packet capture file path
  • {TARGET_IP} — Target IP address
  • {FILE} — Generic file path

5 Guidelines

  • Keep commands copy-paste ready
  • Use realistic placeholder values in examples (e.g., 10.10.10.1)
  • Add a references array with links to official documentation
  • One entry per tool or closely related set of commands
  • Test your commands before submitting
  • Use existing entries as examples for formatting

6 Submit a Pull Request

Commit your changes and submit a pull request with a clear description of what you added:

git checkout -b add-new-tool
git add js/data.js
git commit -m "Add: ToolName - brief description"
git push origin add-new-tool

Then open a pull request on GitHub.

All contributions are reviewed before merging. Make sure your entry follows the format above and contains accurate, tested commands.
